Fig.7 · the hard parts

Where loops die in production. None are prompt problems.

01 TERMINATION

Every loop must be able to give up

Bound everything: max_steps, wall clock, budget, and a no-progress counter.

02 VERIFICATION

Ground truth beats self-report

Wire the check to something unforgeable: a test runner, a type checker, a loudness meter.

03 CONTEXT

Memory is a budget, not a bucket

Summarize, evict, retrieve. What the model sees at step 20 is a design decision.

04 COST

Autonomy multiplies spend

A loop costs steps × tools × retries, unattended, at 3 a.m. Meter and cap it.

05 RECOVERY

Actions fail, make them idempotent

Design actions to be safely retryable. Checkpoint state so the loop resumes, not restarts.

06 BLAST RADIUS

Constrain what a wrong step can touch

Sandbox, least-privilege tools, and a human gate on irreversible moves.

Stop prompting, start building loops
